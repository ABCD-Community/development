Challenging the concepts

● If 'information discovery' means leaving the challenge of finding information (the needles in the haystack ??) to the users themselves, are then librarians not abdicating their real mission ? => information exploration rather than discovery ?

● If information literacy includes the concept of (personal) information management, it becomes a fully different story => information management skills are needed

The ABCD Site

● ABCD Site = Virtual Health Library Site http://www.bireme.br

● ABCD Site as a portal : guiding users to a multitude of field- specific resources, e.g. health

● 3 types of resources (columns) : – External links, e.g. EBSCO Search – Direct links to databases, e.g. local and remote catalogs – Other HTML-based links, e.g. social media, Google Scholar, RSS etc.

=> ABCD was conceived as a discovery-tool more than a 'library automation system'

http://www.bireme.br/

ABCD Site local databases

● See also : the e-Granary project for relevance of local resources ● Library catalog in MARC21 format ● Documentation Centre / Archive / Museum catalogs ● Local scholarly databases converted from (freely) available

data-sets, e.g. – DOAJ (demo conversion) – TEEAL, ASFA, AGRIS... – Factual information (experts, projects, meetings...)

● Local repositories with Full Text (-links)

ABCD Site external databases

● Databases in ISIS, e.g. – DIREVE : events directory – SciELO : health publications – MEDLine : medical abstracts

● Databases of non-ISIS origin which can be accessed through a URL for 1) linking 2) getting a search result and 3) displaying a search result, to be configured in the ABCD Site Admin module (data to be given by the actual database-managers)

ABCD Digital Library features

● New in v2.0 (in test-phase) ● Uses expanded version of ISIS ($cisis_ver) ● Tool for personal information management, e.g.

– 'dark archives' (example of Cuban scholars, document stores of researchers...)

– Personal repository of publications – Institutional repositories

● Batch import or interactive adding to collection (demo) ● DL databases can be managed from ABCD Central and searched

from the OPAC like any database

ABCD databases on discovery/literacy resources

● ABCD allows creation of local databases with ANY structure, e.g. – DC.metadata – URL for original document – DocText field for full-text indexing (using TIKA)

● An 'HTML-area' text field can contain most if not all HTML- formatted information about a resource

● Databases for discovery tools can be created - demo ● Databases for literacy training can be created ● These databases can be included into the Site meta-search

- demo

Making your ISIS-database indexable for crawlers

● The ISIS Print Formatting Language allows adding the necessary '<head> … </head>' statements, e.g. <meta http-equiv="keywords" name="keywords" content="descriptor1, descriptor 2, ..." />

● A report of (a range or all) records in the database can be produced using that PFT using HTML tags as a HTML file.

● Put that file on your server in an indexable directory

Conclusions

● ABCD, as a free tool, can facilitate libraries to : – Pre-select resources (exploration vs. discovery) in the Site – Act as a portal to external resources – Offer searching (meta-search) in a multitude of local and

external databases ● ABCD can also facilitate local management of

document stores ● ABCD can facilitate the management of discovery and

literacy training tools
