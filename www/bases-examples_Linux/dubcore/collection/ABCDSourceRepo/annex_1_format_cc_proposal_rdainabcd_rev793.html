1. Description of the project context (max. 3 pages)

1.1. Background of the project

Original idea and steps undertaken to elaborate the idea. In 2013 a new international standard for cataloging rules was brought into practice by the main libraries, e.g. Library of Congress and British National Library : RDA (Resource Description and Access). This new standard incorporates elements of the ‘FRBR’ model (Functional Requirements for Bibliographic Records), which is a modernisation on the MARC-model which introduces more flexible retrieval based on a hierarchical model where intel - lectual works have ‘expressions’ (e.g. Formats and languages) and ‘manifestations’ (e.g. Edtions). In other words : modern catalogues should be capable of showing all records related to a certain ‘work’ regardless of their expression and manifest - ations. Since the ABCD-system, like all modern more advanced library systems, uses a protocol ‘Z39.50” for ‘copy cataloging’ (or ‘shared cataloging’), which means the complex bibliographic records in MARC-format can be freely copied from the cata- logs of the main international libraries (e.g. LC), the incoming records nowadays are based on the RDA-rules and therefore contain extra information. This extra information at this moment is not correctly dealt with in the catalogs : - the worksheets (with listings of fields) do not contain the new fields - the indexing mechanism (FST) does not take into account the new possibilities of end-user retrieval based on the ‘work’- level of a bibliographic record - the display formats do not display the new information or give links to the alternative display formats (e.g. all works from the same author). This problem was discussed with participants of the ITP Lib@Web 2014 and VLIR/UOS December 2014 workshop parti- cipants lead by Piet De Keyser and the need for an update of the ABCD system became very much articulated. So, with the current ABCD-system, the cataloging is foregoing the new cataloging rules and the ABCD-based library systems do not offer any longer internationally accepted bibliographic records. The alternative solution of ‘downgrading’ RDA to AACR2, while mostly possible from a technical point of view and practiced in some cases (eg. The Flemish common or central catalog for public libraries), is not recommended to be sustained, as in- formation is lost and new exciting features are omitted. For this reason the project proposes to update the ABCD-system to incorporate the RDA-rules and FRBR-principles and pro - mote its use by training the librarians in the VLIR/UOS network to use the new standard.

The benefits for the VLIR/UOS partners are obvious : having at their disposal a library system which serves the newest in - ternational cataloging standards, having trained librarians to use it. For VLIR/UOS there is one additional benefit : as the initiative-taking initial funding body of the ABCD-system development, this project will allow the ABCD system to keep its high esteem also academically as a modern, updated and fully capable library system.

History of the collaboration / antecedents Antecedents – please indicate whether this project proposal is a continuation of an existing project, actions already under- taken, or a project in finalisation phase. Describe that project in 10 lines.

The ABCD system was developed with VLIR/UOS support in the 'DOCBIBLAS' project during 2008-2012. The ABCD system was approved by the VLIR/UOS ITS Experts group as an effort to streamline the support for library automation, avoiding ex- pensive licenses and commercial provider dependencies. It is based on proven technology (ISIS databases, supported by several UN branches, in the Virtual Health Library) along with specific Southern requirments : high self-reliance degree for librarians, full Open Source and community support, academic bibliographic standards. The system was presented by an in- ternational VLIR/UOS workshop in March 2009. Today about 12 VLIR/IUC partner institutions use the software in East- Africa and Latin-America and in Cuba the whole ICT-network library-component is ABCD-based. The ABCD system is an important training component also in the ITP 'Lib@Web' (Univ. Antwerpen) since 2012 and many library-project missions for IUC-projects give training on ABCD. In 2015 UNESCO supports the development of a UNICODE version of ABCD (Amharic version for Ethiopia) to promote its use in localized environments.

2. Description of the project (Max. 2 pages) Consistent with the Logical Framework Matrix (LFM) and the Indicative Operational Plan (IOP)

Description of the project. (max 15 lines) The project will : 1. develop the new RDA-standard technically into the ABCD system : the new RDA-fields, which come on top of the existing MARC-fields, need to be defined and implemented into the MARC-standard and AACR-rules, creating a new model-data - base with adapted RDA 'Field Definition Table'; these new fields need to be incorporated also in the indexing mechanism of the ISIS-database tecnology (the Field Selection Table) and the Print Format Tables, allowing e.g. retrieval of 'related' biblio- graphic entities (publications by same author, on same topic, other manifestations of same work etc.). All this needs to be documented in the existing ABCD-documentation. 2. train the ABCD-librarians into the use of the new standard, by explaining its FRBR-philosophy and AACR cataloging rules and showing how and where ABCD allows to take these fully into account. This training will be given in 2 separate work - shops of one week, one in East-Africa and one in Latin-America.

Describe the developmental relevance of the project. Development work always needs to make sure the tools provided for development narrow the gap with the developments in the Western world, not widen them. So also the library systems need to keep in touch with important innovations in the cataloguing and bibliographical profession in the West. Better, updated catalogues facilitate better and more up-to-date re - search and education in Higher Education.

The ABCD system can only survive if it keeps in touch with international information science developments and standards. As long as this is indeed the case, the ABCD libraries will need to maintain their systems and have their library staff trained on these new skills and standards.
